# Phase 04 — Report

Goal: render the drift report body in Markdown, append hand-off suggestions,
flip status to `reported`.

## Step 1 — Read final frontmatter

Read the drift report's frontmatter. You'll need:

- `scope_mode`, `scope_targets[]`, `parsed_targets[]`, `drifts[]`,
  `probe_metadata`, `phase_history`, `parse_warnings[]`, `probe_notes[]`,
  `skipped_projects[]`.

## Step 2 — Render the body using the template

Read `~/.claude/skills/a1-reconcile/templates/drift-report-template.md` and
fill it. The body must have these sections in order:

1. **Summary** — counts by class, status, target list
2. **Drifts** — grouped by class (MISSING → EXTRA → DIVERGED → STALE), each
   entry as a Markdown subsection
3. **In-Sync confirmations** — short bullet list of artifacts that probed
   clean (one line each)
4. **Skipped projects** (vault-sync only) — slug + reason
5. **Phase history** — table from `phase_history[]`
6. **Suggested next** — hand-off proposals, see Step 3

Use the inline-code style for paths and FR-IDs. No emoji. No "Generated by"
footers — those live in the frontmatter.

## Step 3 — Compute hand-off suggestions

Walk `drifts[]`:

| If the report contains... | Suggest |
|---|---|
| any `MISSING` | `a1-fix` (if narrow) or `a1-new-feature` (if whole feature gap) — list affected FRs |
| any `EXTRA` | `a1-new-feature` Phase 3 to extend the spec, or `a1-fix` to remove the orphan code |
| any `DIVERGED` | Manual inspection, then `a1-fix` |
| any `STALE` | Re-run `a1-reconcile` after the next code change touches the area, or trigger a fresh `a1-analyze` if structural |
| only `IN_SYNC` | Confirm clean state. Suggest next routine: weekly `vault-sync` |

Write the suggestions into the body's "Suggested next" section AND into
frontmatter `suggested_next[]` so other skills can read them. Use this CLI
call (via update-status with phase-data) to set them:

```bash
node ~/.claude/skills/_shared/a1-tools.cjs reconcile update-status \
  "<drift-path>" reported \
  --phase-data '{"suggested_next": [{"skill": "a1-fix", "reason": "...", "targets": ["FR-002", "FR-005"]}]}'
```

## Step 4 — Write the body atomically

The skill writes the rendered Markdown body via the `Write` tool, replacing
the current body (the CLI's `init` produced an empty skeleton). The
frontmatter is preserved verbatim from the previous step — do not touch it
with Write/Edit; only `a1-tools reconcile update-status` is allowed to
modify frontmatter.

To keep the operation safe:

1. Read the file.
2. Split off the frontmatter (lines between the first `---` and the second).
3. Write the file with the unchanged frontmatter block + the new body.

## Step 5 — Final status flip

If Step 3's `update-status reported` call already ran, Step 5 is a no-op.
Otherwise:

```bash
node ~/.claude/skills/_shared/a1-tools.cjs reconcile update-status \
  "<drift-path>" reported
```

## Step 6 — Summarize for the user

> "Drift report complete: `<path>`
>
>  Findings: <n> drifts (<m> MISSING, <e> EXTRA, <d> DIVERGED, <s> STALE),
>  <i> artifacts in-sync.
>
>  Suggestion(s):
>  - <skill>: <reason> — FRs <list>
>  - ...
>
>  Should I trigger one of these, or is the report sufficient?"

Wait for the user's decision. **Never auto-trigger another skill.**

## Edge cases

- **Zero drifts:** still produce a clean report (it's the proof-of-no-drift
  artifact). Frontmatter `drifts[]` is `[]`, body shows "All targets in
  sync." Suggested next: weekly `vault-sync`.
- **Phase 3 partially failed (one agent timed out):** the report body must
  include a "Probe coverage" note in the Summary section listing which
  agents completed and which didn't.
- **vault-sync with skipped projects:** the body's "Skipped projects" section
  must list each one. Hand-offs are computed only over non-skipped targets.
- **User closes the chat before approving Step 6's hand-offs:** the report
  is final on disk (status `reported`), `suggested_next[]` is set in
  frontmatter, no irreversible action was taken. Safe to abandon.

## Retro (mandatory, every run)

After every run — pass or fail — write one structured entry. Takes 2 minutes. Do not skip.

**To local cache:**
```bash
cat >> ~/.claude/skills/a1-reconcile/_learning.md <<'EOF'
---
date: <YYYY-MM-DD>
task: <short description of what was reconciled>
project: <project-slug>
result: <pass|fail|partial>
issues: [<relevant tags: false_missing, false_extra, agent_json_failure, spec_parse_gap, repo_path_missing, stale_misclassified, ...>]
what_worked: <one sentence>
one_line_learning: <what would have prevented the main issue, or "no issues">
EOF
```

**To Vault:**
Append the same entry to:
`~/Documents/Obsidian Vault/areas/a1-learnings/a1-reconcile.md`

A run with no issues is still useful data — write the entry.
